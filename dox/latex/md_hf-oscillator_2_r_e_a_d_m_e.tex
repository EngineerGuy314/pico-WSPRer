\chapter{Digital controlled radio frequency oscillator for Raspberry Pi Pico}
\hypertarget{md_hf-oscillator_2_r_e_a_d_m_e}{}\label{md_hf-oscillator_2_r_e_a_d_m_e}\index{Digital controlled radio frequency oscillator for Raspberry Pi Pico@{Digital controlled radio frequency oscillator for Raspberry Pi Pico}}
\label{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md0}%

 The library for Raspberry Pi Pico includes the headers and source code and all necessary build files to build a custom application which turns pico into precise PLL digital frequency oscillator of the whole of HF radio spectrum (1 Hz to 32.\+333 MHz) with millihertz resolution.\hypertarget{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md1}{}\doxysection{\texorpdfstring{Precise frequency resolution}{Precise frequency resolution}}\label{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md1}
The library provides about 23 milli-\/\+Hz frequency resolution. This resolution is limited by 24-\/bit register which is used in algorithm. The working WSPR beacon which has been built on the base of this project proves that the quality of generated signal is sufficient to such precise (\texorpdfstring{$\sim$}{\string~}1.46 Hz step) frequency manipulation digital modes. The upper freq. limit is \texorpdfstring{$\sim$}{\string~}32.333 MHz and it is achieved only using Pico overclocking to 270MHz.



Here is an example of narrowband FSK (9.\+4 MHz carrier, 5 Hz step, 20 Hz range in total).\hypertarget{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md2}{}\doxysection{\texorpdfstring{Phased locked loop in C}{Phased locked loop in C}}\label{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md2}
The DCO uses phase locked loop principle programmed in C.\hypertarget{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md3}{}\doxysection{\texorpdfstring{{\itshape NO} additional hardware}{{\itshape NO} additional hardware}}\label{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md3}
The DCO provides the output signal on the GPIO pin. However if you want to transmit the signal, you should calculate and provide a lowpass filter of appropriate frequency. Please also figure out whether you possess rights to emit radio frequency energy on desired frequency.\hypertarget{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md4}{}\doxysection{\texorpdfstring{GPS reference frequency correction (optional) since v.\+0.\+9}{GPS reference frequency correction (optional) since v.\+0.\+9}}\label{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md4}
GPS reference frequency correction option provides an absolute frequency error within about \texorpdfstring{$\sim$}{\string~}1\+Hz in long term.

\hypertarget{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md5}{}\doxysection{\texorpdfstring{Dual-\/core}{Dual-\/core}}\label{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md5}
The DCO uses extensively the secodary core of the pico. The first one is for your ideas how to modulate the DCO to obtain a desired signal. The DCO does {\itshape NOT} use any floating point operations -\/ all time-\/critical instructions run in 1 CPU cycle.\hypertarget{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md6}{}\doxysection{\texorpdfstring{Radio transmitters}{Radio transmitters}}\label{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md6}
Owing to the meager frequency step, it is possible to use 3, 5, or 7th harmonics of generated frequency. The practical resolution will be quite the same -\/ far below 1 Hz. Such solution completely cover all HF and VHF band up to \texorpdfstring{$\sim$}{\string~}233 MHz.\hypertarget{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md7}{}\doxysection{\texorpdfstring{Console}{Console}}\label{md_hf-oscillator_2_r_e_a_d_m_e_autotoc_md7}
